## Full-Stack Architect

### Key Recommendations
1. **SSE (Server-Sent Events)** — lightweight real-time updates without WebSocket complexity
   - Python server watches file changes → pushes JSON patches via SSE
   - Dashboard JS receives patches → updates DOM incrementally
2. **Keep file-based state** but add `StateManager` abstraction layer
   - All reads/writes through one interface
   - Easy to swap to SQLite/Redis later without touching UI code
3. **Split architecture**: static HTML shell + dynamic JS modules
   - HTML generated once (structure), JS fetches data and renders
   - Enables partial updates without full rebuild
4. **Action whitelist** — define safe operations (restart service, change mode, mark task done)
   - Each action = POST to local API with explicit validation
   - No arbitrary command execution
5. **Plugin system** — each dashboard section as independent module
   - `register('codexbar', { fetch, render, actions })`
   - Easy to add/remove sections

### Priority Actions
1. Local API server (FastAPI/Flask, ~100 lines) on localhost:7777
2. SSE endpoint for live updates
3. Action endpoints: restart-service, change-mode, mark-done

## Consensus (all 4 agree)

1. **"Since last visit" diff** — the single most impactful feature; every expert recommended it as #1 priority
2. **Staleness detection** — dashboard must warn when data is stale (all agreed current static page gives false confidence)
3. **Quick actions** — dashboard should DO things, not just show things (restart services, change mode, mark done)
4. **Exception-based display** — hide healthy/OK items, surface problems first
5. **Local API server** — needed for any interactivity; lightweight Python server on localhost:7777
6. **Remove/collapse low-value sections** — Network graph, Done projects → collapsed or removed
